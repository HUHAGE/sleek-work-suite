# 暗黑模式卡片优化文档

## 优化概述

针对暗黑模式下各个菜单功能页面中的卡片不够明显的问题，进行了全面的视觉优化，提升了暗黑模式下的结构清晰度和视觉层次。

## 优化内容

### 1. 全局样式优化 (`src/index.css`)

#### 暗黑模式颜色变量调整
- **card**: `0 0% 3.9%` → `0 0% 10%` (提升卡片背景亮度)
- **popover**: `0 0% 3.9%` → `0 0% 8%` (提升弹出层背景亮度)
- **muted**: `0 0% 14.9%` → `0 0% 18%` (提升次要背景亮度)
- **border**: `0 0% 14.9%` → `0 0% 20%` (增强边框可见度)
- **input**: `0 0% 14.9%` → `0 0% 18%` (提升输入框背景亮度)

#### tool-card 类优化
```css
.tool-card {
  /* 暗黑模式下的改进 */
  - 背景透明度: 50% → 80% (更实在的背景)
  - 边框: 保持原有 → 完整边框 (更清晰的边界)
  - 阴影: 无 → 添加阴影效果 (增强层次感)
  - 悬停效果: 优化边框和阴影的过渡
}
```

  - 添加悬停时的阴影效果

  - 暗黑模式使用完整边框颜色

## 优化效果

### 视觉改进
1. **更清晰的边界**: 通过增强边框宽度和透明度，卡片边界更加明显
2. **更好的层次感**: 通过添加阴影效果，增强了卡片的立体感
3. **更高的对比度**: 通过提升背景亮度，文字和内容更易阅读
4. **更流畅的交互**: 优化了悬停效果，提供更好的视觉反馈

### 一致性提升
- 所有卡片组件使用统一的边框宽度 (`border-2`)
- 统一的暗黑模式背景透明度 (80%)
- 统一的阴影效果
- 统一的悬停交互效果

## 技术细节

### CSS 变量使用
- 使用 Tailwind 的 `dark:` 前缀实现暗黑模式样式
- 使用 HSL 颜色空间便于调整亮度
- 使用透明度控制实现玻璃态效果

### 性能考虑
- 所有样式使用 Tailwind 类，编译时优化
- 过渡动画使用 GPU 加速属性
- 避免使用复杂的 CSS 滤镜
